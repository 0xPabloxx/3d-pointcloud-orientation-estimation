# 文档目录

本目录存放所有项目相关的分析、实验报告和方法论文档。

## 📁 目录结构

```
docs/
├── README.md                   # 本文件
├── analysis/                   # 问题分析、调试诊断
├── experiments/                # 实验结果报告
└── methods/                    # 方法论、理论说明
```

---

## 📄 analysis/ - 问题分析文档

**用途**: 记录问题诊断、根因分析、调试过程

**命名规范**: `analysis_YYYYMMDD_<主题描述>.md`

**现有文档**:
- `analysis_20251109_glassbox_training_failure.md` - Glassbox训练失败的根因分析（zeros初始化问题）

**何时创建**:
- 训练失败需要深入分析时
- 发现反直觉现象需要解释时
- 性能瓶颈诊断时
- 数据质量问题调查时

---

## 🧪 experiments/ - 实验结果报告

**用途**: 完整记录实验配置、结果、分析、结论

**命名规范**: `experiment_YYYYMMDD_<实验名称>_results.md`

**现有文档**:
- `experiment_20251109_init_fix_results.md` - 初始化修复实验完整报告（Val Loss 0.74→0.0017）

**何时创建**:
- 完成重要实验后
- 消融实验（ablation study）后
- 对比不同方法/超参数后
- 达到里程碑结果时

**必须包含**:
1. 实验目标和假设
2. 实验配置（超参数、数据、模型）
3. 定量结果（loss曲线、指标表格）
4. 定性分析（可视化、case study）
5. 结论和下一步建议

---

## 📚 methods/ - 方法论文档

**用途**: 详细说明技术方法、算法原理、理论基础

**命名规范**: `method_<方法名称>.md`

**计划创建**:
- `method_mvm_distribution.md` - von Mises混合分布理论
- `method_hungarian_matching.md` - 匈牙利算法在MvM loss中的应用
- `method_rotation_invariance.md` - 旋转不变性与数据增强
- `method_kl_divergence_loss.md` - KL散度loss的推导和实现

**何时创建**:
- 使用了复杂算法需要说明时
- 需要数学推导支撑时
- 为论文写作准备素材时
- 团队知识沉淀时

**建议内容**:
1. 方法背景和动机
2. 数学定义和公式
3. 算法伪代码
4. 代码实现细节
5. 优缺点分析
6. 参考文献

---

## 📝 文档编写规范

详见根目录 `claude.md` 的"Markdown文档命名与存储规范"章节。

**核心要求**:
- ✅ 专业性：适合作为论文参考材料
- ✅ 完整性：任何人读了都能复现
- ✅ 结构化：用标题、列表、代码块、公式
- ✅ 量化：有具体数字、图表、对比
